People keep asking me where to start! - I'm no expert in pedagogy, but regardless here are my recommendations.

An emphasis on freely available Massive Open Online Classes (MOOCs) is apparent.
Start by learning the:
  1. command-line for your OS (Linux [also mostly applicable for Mac]; Windows)
  2. git (for backing up, versioning, and collaborating on source-code)
Also an introduction to operating systems course wouldn't go astray. Now you're ready to start learning software engineering.
Introductory courses
Recommend you start with an introduction to computer science, such as:
  • Harvard (CS50x), and
  • MIT (6.00x)
Then continue onto databases with:
  • Stanford (Databases)
Second-year courses
Now that you understand the basics and have 3+ languages under your belt, I recommend:
  1. SaaS I  from UC Berkeley (CS169-1x)
  2. SaaS II from UC Berkeley (CS169-2x)
  3. Something functional, e.g.: Programming Languages Part C from UW; Functional Programming Principles in Scala from EPA
  4. Data-structure and algorithms courses from Stanford
Final-year courses
Here comes the most important courses, the one which allow you to put everything together and get a little advanced:
  1. Compilers from Stanford (CS1)
  2. Fundamentals of Cloud Computing with Microsoft Azure from Harvard Extensions School (CSCI E-94)

This is also when I would recommend learning Android (Java), iOS (Swift), Angular and Node.JS (JavaScript & TypeScript) development.

Pickup other knowledge you want here also, like: Artificial Intelligence; Natural Language Processing; game physics development &etc.

Literature

Some people learn better from textbooks, regardless they are a good reference; even if you don't read them page-for-page.

  • C
    • C Programming Language, 2nd Edition
    • Deep C secrets
    • Modern C
Finishing

Now that you have pretty much acquired the specialisation, you should be able to understand how any real-world problem can be solved computationally.

You'll also be able to pickout the ones that can't be solved efficiently.
